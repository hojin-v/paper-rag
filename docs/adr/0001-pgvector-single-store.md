# ADR-0001: Vector DB로 별도 스토어 대신 PostgreSQL+pgvector 채택

- 상태: 승인 (2026-07-04)

## 배경

요구사항은 RDB(메타데이터·키워드)와 Vector DB(단락·키워드·표 임베딩)를 논리적으로 분리해 정의했다.
후보: pgvector(통합) vs ChromaDB(별도 스토어).

## 결정

PostgreSQL 16 + pgvector 단일 저장소. 논리적 컬렉션(단락/키워드/표)은 각 테이블의 VECTOR 컬럼 + HNSW 인덱스로 구현.

## 근거

1. 검색 흐름이 "정확 매칭(SQL) → 실패 시 벡터 유사도"로 정형·벡터 질의가 한 흐름에 섞임 — SQL 조인으로 처리하는 편이 구조적으로 단순
2. 엑셀 출력이 단락·표·메타데이터 다중 조인을 요구 — Chroma는 조인 불가라 메타데이터 중복 저장 필요
3. 삭제/수정 시 트랜잭션 일관성 — 별도 스토어는 이중 쓰기 정합성 코드가 추가됨
4. 온프레미스 운영 부담 — 서비스 1개 감소, pg_dump 단일 백업
5. 규모(수십만 단락)에서 HNSW 성능 충분. 초과 성장 시 이관 대상은 Chroma가 아닌 Qdrant/Milvus

## 영향

- Chroma는 임베딩 모델 비교 등 실험 용도로만 허용, 운영 반영 금지
- 스키마의 모든 임베딩 컬럼은 VECTOR(1024) (BGE-M3), 코사인 거리 기준

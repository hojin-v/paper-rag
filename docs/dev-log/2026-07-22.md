## 버그 1: 검색 질의 오역 상세

- **재현**: 저장된 정확 키워드 `"Structured Document Understanding"`을 영어 그대로 질의해도
  질의 키워드 추출 LLM이 이를 깨진 한글(자모 미조합, 예: "구조화되ᄂ 문서 이해")로 오역해
  정확 매칭이 실패하고 유사 후보로 빠졌다(2026-07-22 오전, 2회 재현 — 이전 세션에서 발견).
- **원인**: `search/service.py`의 `QUERY_KEYWORDS_PROMPT`가 한국어로만 작성돼 있고 "원문 언어를
  유지하라"는 지시가 없었다.
- **수정**: 번역 금지 지시 + 영어/한국어 few-shot 예시 2개 추가.
- **검증**: 실제 Ollama에 재현 질의를 다시 던져 `status: matched`로 정상 매칭됨을 확인했고,
  한국어 질의(`레이아웃 인식 방법`)도 회귀 없이 정상 동작함을 확인했다. `tests/test_search_service.py`에
  프롬프트가 번역 금지 지시를 유지하는지 고정하는 회귀 테스트 추가.

## 버그 2: 표 셀 구조 깨짐 상세

- **재현**: LiLT 논문의 표 11개 중 표 1번에서 헤더·데이터 여러 행의 텍스트가 한 셀에 뒤섞여
  있었다(예: `# Inter-modal Operation 1 CAT 2 CAT+Co-Attention (Lu et al., 2019) | | Average F1 0.6751` —
  2번째 열 전체가 사라지고 값이 인접 열에 합쳐짐).
- **원인**: `paddle_backend.py::_TableHtmlParser`가 표 구조 모델(SLANeXt/SLANet_plus)이 내놓는
  `pred_html`의 `colspan`/`rowspan` 속성을 전혀 읽지 않고 `<td>` 등장 순서로만 칸을 채웠다 —
  병합 헤더 셀이 있으면 그 뒤 모든 열이 한 칸씩 밀린다(코드 docstring에 이미 알려진 제약으로
  명시돼 있었음).
- **수정**: colspan만큼 같은 텍스트를 여러 열에 복제하고, rowspan은 `_pending` 딕셔너리에
  등록해 다음 `<tr>` 시작 시 해당 열을 먼저 채우는 방식으로 그리드 위치를 복원. 부수적으로
  셀 값 안의 리터럴 `"|"`(OCR 오인식 등)가 열 구분자로 오인되지 않도록 `"/"`로 치환하는
  이스케이프도 추가.
- **검증**: `tests/test_paddle_mapping.py`에 colspan 헤더/rowspan 셀/리터럴 파이프 3개 케이스로
  재현 테스트 추가. 실제 corrupted 표 1번과 동일한 패턴(헤더 colspan)을 합성 HTML로 재현해
  수정 후 정상적으로 정렬됨을 확인.
- **중요한 한계**: 이미 DB에 적재된 LiLT 논문 표 1번 자체는 **재인제스트 전까지 여전히 깨진
  채로 남아 있다** — 저장 시점에 이미 잘못된 그리드로 직렬화됐으므로 코드 수정만으로는
  복구되지 않는다. 코드 수정은 앞으로 새로 적재되는 문서부터 적용된다. 발표 영상에서는
  표 2번 이후(정상 확인됨)를 사용하도록 `video-script.md`에 명시했다.

# 06. Streamlit 검색 UI

자연어 질의에서 대표/연관 논문 확인과 엑셀 다운로드까지 이어지는 사용자 화면을 실행한다.

```text
사용자
  └─ Streamlit UI
       ├─ POST /search
       │    ├─ matched: 결과 카드 + 엑셀 다운로드
       │    └─ suggest: 유사 키워드 3개 선택
       └─ POST /search/select
            └─ matched: 결과 카드 + 엑셀 다운로드
```

# 1단계: UI 의존성 설치

| 항목 | 값 | 설명 |
| --- | --- | --- |
| optional extra | `ui` | Streamlit 실행 의존성 |
| 설치 명령 | `pip install -e ".[ui]"` | 개발 환경에서 패키지를 editable로 설치 |

```bash
pip install -e ".[ui]"
```

검증:
```bash
python -c "import streamlit"
python -c "import paperrag.ui.client"
```

# 2단계: API 서버 기동

| 항목 | 값 | 설명 |
| --- | --- | --- |
| API 앱 | `paperrag.search.api:app` | UI가 호출하는 FastAPI 엔트리포인트 |
| 기본 URL | `PAPERRAG_API_BASE_URL=http://localhost:8000` | Streamlit UI의 API 접속 주소 |

```bash
uvicorn paperrag.search.api:app --host "${PAPERRAG_API_HOST:-0.0.0.0}" --port "${PAPERRAG_API_PORT:-8000}"
```

검증:
```bash
curl -s http://localhost:8000/health   # {"status":"ok"}
```

> 주의: UI는 검색 API 경유로만 동작한다. Streamlit 코드에서 DB나 리포지토리에 직접 접근하지 않는다.

## 2-1단계 (선택): 데모 모드 — DB·LLM 없이 UI 확인

PostgreSQL·Ollama·임베딩 서버 없이 화면 흐름만 확인할 때는 시드 데이터가 내장된 데모 서버를 쓴다.

```bash
python scripts/demo_server.py --port 8000
```

| 데모 질의 | 기대 동작 |
| --- | --- |
| `스마트팩토리 이상탐지 논문` | 정확 매칭 → 대표(논문10)+연관(논문20) + 엑셀 다운로드 |
| `예지보전 논문` | 유사 키워드 3개 제안 → "예측 유지보수" 선택 → 결과 |

> 데모 서버는 인메모리 시드 데이터로만 응답한다. 운영 검증은 반드시 실제 스택(02·03 문서)으로 수행.

# 4단계: 화면 흐름 확인

정확 매칭 경로는 질의 입력 후 바로 결과 카드와 엑셀 다운로드 버튼을 보여준다.

| 화면 요소 | 내용 |
| --- | --- |
| 질의 입력 | 자연어 검색 문장 |
| 대표 논문 카드 | 제목, 저자, 연도, 저널, 매칭 키워드, 점수, 선정 사유 |
| 연관 논문 카드 | 제목, 저자, 연도, 저널, 매칭 키워드, 점수, 선정 사유 |
| 엑셀 다운로드 | `result_id`로 API에서 받은 `.xlsx` 바이트 |

유사 선택 경로는 정확 매칭 실패 시 후보를 고른 뒤 결과를 보여준다.

| 화면 요소 | 내용 |
| --- | --- |
| 유사 키워드 radio | 키워드와 유사도 표시, 최대 3개 |
| 이 키워드로 검색 | `session_id`와 `keyword_id`로 `/search/select` 호출 |
| 결과 카드 | 선택 키워드 기준 대표/연관 논문 |
| 엑셀 다운로드 | 선택 결과의 `result_id`로 받은 `.xlsx` 바이트 |

검증:
```bash
pytest tests/test_ui_client.py -q
```

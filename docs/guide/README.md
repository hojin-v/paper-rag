# 구현 가이드 문서 규칙

모든 가이드 문서는 아래 형식을 따른다. (참고 스타일: NCP 인프라 구축 가이드 — 단계별·표 중심·검증 포함)

## 파일 규칙

- 문서 하나 = 하나의 구축 주제, 파일명 `NN-주제.md` (NN은 진행 순서)
- 최상단: 한 줄 목표 + 전체 구성을 보여주는 코드블록(디렉터리 트리 또는 다이어그램)

## 본문 형식

- `# N단계: 제목` 순서로 진행 (0단계부터 가능)
- 설정값·선택지는 **표**로 정리: `| 항목 | 값 | 설명 |`
- 실행 명령은 코드블록으로, 바로 복사-실행 가능하게
- 각 단계 끝에 **검증 방법** (실행 결과 확인 명령) 포함
- 주의사항·비용·보안·비가역 작업은 `>` 인용문으로 강조

## 문서 끝

- `## 완료 체크리스트` — 이 문서를 끝냈을 때 확인 가능한 상태 목록 (체크박스)

## 예시 뼈대

```markdown
# NN. 주제

한 줄 목표.

(전체 구성 코드블록)

# 1단계: ...

| 항목 | 값 |
| --- | --- |
| ... | ... |

​```bash
명령
​```

검증:
​```bash
확인 명령   # 기대 결과
​```

> 주의: ...

## 완료 체크리스트
- [ ] ...
```

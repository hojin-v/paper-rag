# 발표자료 구성

이 폴더엔 **두 세대**의 자료가 섞여 있다 — 실제 발표·촬영에 쓰는 최종본과, 그 이전에 만든
Marp 소스(구조·초안 참고용)다. 실제 발표/촬영은 **최종본**(`발표자료.pptx`/`영상발표자료.pptx`
+ 각 대본)을 기준으로 한다.

## 최종본 (실제 발표·촬영에 사용)

| 파일 | 용도 |
| --- | --- |
| `발표자료.pptx` | 라이브 발표용 20장 슬라이드(최종본) |
| `발표대본.md` | `발표자료.pptx` 전용 발표자 대본 + 슬라이드별 예상 Q&A(약 9~10분) |
| `EVALUATION_NOTES_AND_QA.md` | 성능 평가 파트(14·15번 슬라이드) 전용 심화 대본·Q&A·프레이밍 규칙 — 평가위원이 가장 깊게 파고드는 구간이라 별도로 두껍게 정리 |
| `영상발표자료.pptx` | 5분 프로젝트 소개 영상용 12장 슬라이드(최종본) |
| `영상발표대본.md` | `영상발표자료.pptx` 전용 내레이션 대본 + 촬영·편집 노트 |

## Marp 소스 (구조 설계 초안 — 슬라이드 장수·구성이 최종본과 다름)

`PRESENTATION.md`는 **8단계 구조**(①기획개요 ②생성형AI핵심기술 ③아키텍처·설계도 ④오픈소스화
⑤프로젝트 핵심 체크포인트 ⑥형상관리 ⑦실시간 솔루션 시연 ⑧향후 발전방향·소회)를 따르는 25장
분량의 Marp 슬라이드 소스다. 최종본(`발표자료.pptx`, 20장)으로 다듬어지기 전 단계의 구조 설계
참고 자료로 남겨두되, **최종 발표에는 쓰지 않는다**. 다만 성능 평가 수치처럼 두 세트가 공유하는
사실 정보는 어긋나지 않게 함께 갱신한다(2026-07-24, 15편 파일럿 실측치 반영).

| 파일 | 용도 |
| --- | --- |

반영한다(과거의 kiwipiepy/LLM 이중 토글은 제거됨). 구버전 어두운 배경 세트는 더 이상 참조되지 않는다.

## 사용 원칙

- 발표 화면에는 **`발표자료.pptx`(라이브)/`영상발표자료.pptx`(영상)만** 사용한다. `PRESENTATION.md`는
  화면에 띄우지 않는다(Marp 소스, 구조 참고용).
- 긴 설명은 슬라이드에 추가하지 않고 `발표대본.md`(성능 평가는 `EVALUATION_NOTES_AND_QA.md`)에서
  발표자가 말한다.
- 평가자가 기능 범위, 모델 근거, CPU 가능성, 품질 검증을 질문하면 해당 슬라이드의 Q&A를 사용한다.
- 현재 구현과 다음 목표를 섞어 말하지 않는다. `현재와 목표·우선순위` 슬라이드에서 명시적으로 구분한다.
- 성능 수치는 **항상 "n=15 파일럿 예비값"으로 표기**하고 확정 성과처럼 말하지 않는다
  (`EVALUATION_NOTES_AND_QA.md` §0/§2의 프레이밍 규칙 참고).

## 렌더링

`PRESENTATION.md`는 Marp 형식이다. Marp를 지원하는 편집기에서 열면 16:9 슬라이드로 발표하거나
PDF/PPTX로 내보낼 수 있다. Marp가 없어도 일반 Markdown 문서로 내용과 이미지를 확인할 수 있다.

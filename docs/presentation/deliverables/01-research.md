# 자료조사 — 온프레미스 논문 분석 RAG

프로젝트 설계 단계(2026-07-04)와 이후 조정(2026-07-12, 2026-07-20)에서 실제로 검토·결정한 내용을
정리한다. 가정이 아니라 `docs/design/DESIGN.md`, `docs/adr/0001~0002`에 기록된 결정과 근거를 그대로 옮긴다.

## 1. 문제 정의

- 논문 PDF는 페이지마다 레이아웃·품질이 다르고, 본문·표·참고문헌이 뒤섞여 있어 그대로 검색에 쓸 수 없다.
- 자동 추출 결과를 신뢰하기 어렵다 — 사람이 확인할 수 있는 검수 과정이 필요하다.
- 목표: 추출 과정을 투명하게 보여주고, 검색 결과를 엑셀로 바로 활용 가능하게 한다.

## 2. 문서 구조 분석(레이아웃·OCR) 모델 비교

| 후보 | 라이선스 | 검토 결과 |
| --- | --- | --- |
| **PaddleOCR PP-StructureV3** (PP-DocLayout, PP-OCRv5, SLANeXt) | Apache 2.0 | **채택**. 디지털·스캔 PDF에 동일 경로 적용 가능, PaddleX로 파인튜닝 공식 지원, CPU 실행 확인 |
| Docling | MIT | 진단·비교 용도로만 사용. 초기(2026-07-04)에는 디지털 PDF 트랙으로 병행 채택했으나 2026-07-12 결정으로 운영 경로에서 제외(§4 참고) |
| LayoutLMv3 / DiT | CC BY-NC | **배제** — 가중치가 비상업 라이선스 |
| Surya / Marker | GPL + 가중치 상용 조건부 | **배제** |
| MinerU | AGPL-3.0 | 벤치마크 baseline으로만 사용, 운영 대체 경로 아님 |
| DocLayout-YOLO | AGPL-3.0 | 배제, 성능 백업 후보로만 보류 |

## 3. 저장소 비교 — RDB+Vector 통합 vs 분리

| 후보 | 검토 결과 |
| --- | --- |
| **PostgreSQL 16 + pgvector** | **채택**(ADR-0001). 검색 흐름이 "정확 매칭(SQL) → 실패 시 벡터 유사도"로 정형·벡터 질의가 한 흐름에 섞이고, 엑셀 출력이 단락·표·메타데이터 다중 조인을 요구하므로 단일 저장소가 구조적으로 단순 |
| ChromaDB(별도 벡터 스토어) | 배제 — 조인 불가로 메타데이터 중복 저장 필요, 이중 쓰기 정합성 코드 추가, 운영 서비스 1개 증가 |
| Qdrant/Milvus | 향후 규모 초과(수십만 단락 이상) 시 이관 후보로만 기록 |

## 4. 파싱 전략 변경 이력 (이중 트랙 → 단일 경로)

- 2026-07-04: ADR-0002로 **디지털 PDF=Docling / 스캔 PDF=PP-StructureV3** 이중 트랙 채택, Phase 0에서
  50편 실측(mAP/CER/TEDS/E2E F1) 후 최종 확정 예정이었음.
- 2026-07-12: 사용자 결정으로 디지털 파싱(Docling) 운영 경로를 폐기하고, **모든 PDF를 페이지 이미지로
  변환 후 PP-StructureV3로 전체 OCR**하는 단일 경로로 통합(`docs/design/DESIGN.md` 50번째 줄 참고).
  이유: 텍스트 레이어 품질 편차로 두 경로 간 결과 일관성을 보장하기 어려웠음.
- ADR-0002 문서 자체는 과거 결정 기록으로 보존하고, 실제 운영 기준은 DESIGN.md가 최신본이다.

## 5. PDF 렌더링 라이브러리 교체

- 최초: PyMuPDF — AGPL-3.0 라이선스 문제로 2026-07-20 교체
- 교체 후: **pypdfium2(Apache-2.0/BSD-3-Clause) + pypdf + pdfplumber** — 텍스트 레이어는 본문 추출에
  쓰지 않고 페이지 이미지화에만 사용하므로 라이선스 제약이 가벼운 조합으로 대체 가능했다.

## 6. 임베딩·LLM 모델 비교

| 역할 | 채택 | 근거 |
| --- | --- | --- |
| 임베딩 | BGE-M3 (sentence-transformers, 1024차원) | 한/영 혼용 문서 검색에 적합, CPU 실행 가능 |

## 7. 데이터 수집원 조사

| 우선순위 | 수집원 | 비고 |
| --- | --- | --- |
| 1 | 사내/기관 보유 PDF | 일괄 반입, 저작권 리스크 최소 |
| 2 | KCI(OAI-PMH), OAK | 언어·공개 여부 집계 후 원문 링크·CCL 확인 |
| 3 | Europe PMC, CORE, CC 라이선스 arXiv, DOAJ | 공식 API/벌크 경로, 논문별 라이선스 필터 |
| 4 | DBpia 등 유료 DB | 구독 약관 범위 내 수동 확인만, 크롤링 금지 |

- 한국어 CC 라이선스 논문 표본이 매우 적음을 실측으로 확인(OpenAlex `language:ko` 검색 총 439건 수준,
  AI/OCR 주제로 좁히면 거의 없음) — `docs/guide/14-evaluation-labeling.md` 참고.
- 원문 단락을 엑셀로 재출력하므로 저작권 검토(연구 목적 이용 범위, 유료 DB 약관의 TDM 허용 여부,
  출처 표기 방식)가 법무 확정 전제 조건으로 남아 있다.

## 8. 결론

사전학습 모델 조합(PP-StructureV3 + BGE-M3 + Qwen2.5-7B)으로 우선 기능을 완성하고, 정확도는
"측정 없이 튜닝 없다"는 게이트 방식으로 개선한다(`DESIGN.md` §6). 자체 평가셋 없이는 파인튜닝
여부를 판단하지 않는다.

# 평가지표 자료 — 최종 발표용

**원칙**: `DESIGN.md` §6 "측정 없이 튜닝 없다." 측정되지 않은 수치는 발표에서 확정값처럼 제시하지 않는다.
이 문서는 목표 합격선과 **현재까지 실제로 확인된 것**만 구분해서 정리한다. 성능 평가 슬라이드 전용
심화 대본·Q&A·프레이밍 규칙은 `../EVALUATION_NOTES_AND_QA.md`를 함께 본다(이 문서와 수치가
어긋나지 않도록 함께 갱신한다).

## 1. 목표 합격선 (DESIGN.md §6) vs 15편 파일럿 예비 측정 (2026-07-24)

| 대상 | 지표 | 합격선 | 파일럿 예비값(n=15) | 판정 |
| --- | --- | --- | --- | --- |
| 레이아웃 분류 | mAP@0.5 | ≥ 0.85 | ≈ 0.71 | 🔴 미달(병목) |
| 읽기 순서 | 단락 순서 일치율 | ≥ 0.95 | ≈ 0.94 | 🟡 근접(오차범위 추정) |
| OCR | CER(LLM 정제 후 잔존 기준) | ≤ 3% | ≈ 1.8% | 🟢 충족 |
| 표 구조 | TEDS | ≥ 0.85 | ≈ 0.80 | 🟡 근접(재측정 시 상승 가능성 — §2 참고) |
| E2E | 단락 추출 F1 | ≥ 0.90 | 미측정 | ❌ 별도 종단 평가셋 필요 |

**중요**: 이 표의 파일럿 값은 **n=15 소규모 예비 측정**이지 정식 지표가 아니다. 정식 지표는
50~100편 층화 샘플(디지털/스캔·1단/2단·한/영·스캔 품질 축)로 재측정한 뒤 확정한다. 발표에서는
절대값을 확정 성과로 말하지 않고 "경향 + 원인 + 개선 계획 + 방법론"으로 프레이밍한다(상세 스크립트는
`EVALUATION_NOTES_AND_QA.md` §1~§2).

**모델 신뢰도 vs 라벨 대조 지표의 갭**: 레이아웃은 모델 자체 신뢰도가 0.93으로 높게 나오지만, 사람
라벨과 대조한 mAP는 0.71로 합격선에 못 미친다 — "확신도가 높다"와 "정확하다"는 다른 축임을 보여주는
핵심 근거. 원인은 소표본, 그림·수식·참고문헌 제외 경계의 사람 판단 편차, 학술 2단 조판이 공개
벤치마크 분포와 다른 도메인 갭(자세한 원인 분석은 `EVALUATION_NOTES_AND_QA.md` §3 Q3).

## 3. 골든셋 라벨링 (완료)

`docs/guide/14-evaluation-labeling.md` 기준 영문 10편 + 한글 5편, 총 15편을 정답 라벨링 대상으로
선정해 완료했다. 라벨링은 **레이아웃 정리(오검출 제거)** → **텍스트 원문 대조 교정**(`corrected_text`)
2단계로 구성된다.

### 실제 확인된 정성적 사례 (수치가 아니라 근거로 사용)

| 사례 | 내용 |
| --- | --- |
| 표 구조화 | 표 3개(`b-000141`, `b-000155`, `b-000165`) 모두 confidence 0.9 이상, 열/행 파이프 구조가 원문과 일치 |
| 정상적인 오검출 제외 | arXiv 세로 워터마크가 `VZ0Z\nZASOE\n:AIX`로 깨져 인식됐고, 검수 과정에서 정상적으로 rejected 처리됨 |
| 실제 잔존 오류 발견 | 저자 이메일 블록에서 `Jaiswalofficial2908@gmail.com`이 `Jaiswaloffi cial2908@gmail.com`으로 오인식된 채 텍스트 교정 없이 승인됨 — 대량 승인(`approve-all`) 경로가 이런 오류를 놓칠 수 있음을 보여주는 실측 사례 |
| 한글 OCR 정성 확인 | 완성된 문장 단위는 원문과 거의 일치, 표 제목·키워드처럼 짧고 밀집된 텍스트에서 자모 유사 오인식(`HIV`→`H1V`, `신고`→`전고`) 빈도가 더 높음 (`docs/guide/14-evaluation-labeling.md` §1) |

## 4. 실기동 리허설에서 발견·수정한 버그 (2026-07-22 ~ 2026-07-24)

발표 시연 영상·라이브 데모 리허설을 위해 실제 적재된 논문으로 검색→선택→엑셀 다운로드,
검수→적재 전체 흐름을 라이브로 실행하는 과정에서 발견됐다. "테스트는 통과하지만 실제로 돌려보면
다른 문제"의 실제 사례로, 정성적 근거로만 쓴다(수치 지표 아님) — 동시에 이 프로젝트가 실제로
라이브 테스트를 거쳤다는 근거이기도 하다.

| 버그 | 증상 | 원인 | 조치 |
| --- | --- | --- | --- |
| 검색 질의 오역 | 저장된 정확 키워드("Structured Document Understanding")를 영어로 그대로 질의해도 LLM이 깨진 한글로 오역해("구조화되ᄂ 문서 이해") 정확 매칭 실패, 유사 후보로 빠짐(2회 재현) | 질의 키워드 추출 프롬프트에 "원문 언어 유지" 지시가 없었음 | 번역 금지 지시 + 영어/한국어 few-shot 예시 추가, 재현 질의로 재검증 완료(커밋 `7d85e81`) |
| 표 셀 구조 깨짐 | 표 11개 중 1개에서 헤더·데이터 여러 행의 텍스트가 한 셀에 뒤섞임(예: 2번째 열 전체가 사라지고 값이 인접 열에 합쳐짐) | 표 구조 HTML 파서가 `colspan`/`rowspan` 속성을 읽지 않아 병합 셀 뒤 모든 열이 밀림 | colspan/rowspan을 반영해 그리드 위치 복원(커밋 `bde2321`). 이미 적재된 해당 표 자체는 재인제스트 전까지 여전히 깨진 채로 남아 있다 — 코드 수정은 향후 신규 적재분부터 적용 |
| 표 산출물이 "표 형태"가 아님 | 엑셀 "표 셀" 시트가 (구분/표번호/표제목/행/열/셀값) 평면 목록이라 열어도 표로 안 읽힘 | 원래 설계가 평면 리스트였음 | 표마다 실제 엑셀 표(Table) 객체로 등록(필터·줄무늬 서식 포함)하도록 재구성(커밋 `b7e9581`) |
| 검수 적재가 저널·전문 링크를 안 채움 | OpenAlex로 수집한 논문도 검수 화면으로 적재하면 저널·링크가 항상 빈칸 | 자동 수집 적재 경로만 collection-manifest를 조회하고, 검수 화면 "적재" 경로는 조회 자체를 안 함 | 검수 적재도 원본 업로드 파일명에 남은 OpenAlex ID로 manifest를 조회하도록 배선(커밋 `0023e08`) |
| 섹션 선택이 대표/연관 논문을 구분 안 함 | 산출물 구성의 섹션 멀티셀렉트가 대표+연관 논문 섹션을 하나로 합쳐 보여줘 어느 논문 것인지 혼란스럽고, 한쪽에만 있는 섹션을 고르면 다른 쪽 단락이 조용히 0건이 됨 | 두 논문을 하나의 `section_query`로 필터링 | 대표/연관 논문별로 완전히 독립된 두 위젯으로 분리(커밋 `becbd3b`) |
| LLM 재시도가 temperature=0이라 같은 실패를 반복 | 한국어 언어 오염 검증 실패 시 1회 재시도해도 결정론적 생성이라 같은 입력엔 같은 출력이 나와 항상 다시 실패 | 재시도 시에도 temperature=0을 그대로 사용 | 재시도(attempt>0)에만 temperature를 0.2로 올려 결정론을 깨도록 수정(커밋 `51765cd`) |

## 5. 수치 계산에 남은 작업

- `scripts/export_ocr_evaluation.py`(비교 엑셀 생성)에 더해 CER/mAP/TEDS 수치 계산 로직을 구현해
  §1의 파일럿 예비값을 산출했다. 다만 이 계산 스크립트 자체는 아직 50~100편 규모의 정식 재측정에
  맞춰 검증되지 않았으므로, 정식 지표 확정 전 별도 재검증이 필요하다.
- 표 TEDS는 이번 세션에서 colspan/rowspan 버그를 고쳤지만(§4), 이미 적재된 표 데이터는
  재인제스트 전까지 파일럿 측정치에 버그가 섞여 있다 — 재측정 시 상승 가능성이 있다.

## 6. 발표 슬라이드용 요약 문구

> "15편 파일럿에서 OCR은 CER 1.8%로 합격선(3%)을 충족했지만, 레이아웃은 mAP 0.71로 합격선(0.85)에
> 못 미쳤습니다. 모델 자체 신뢰도는 0.93으로 높은데도 그렇다는 게 핵심입니다 — 확신도가 정확도를
> 보장하지 않는다는 뜻이죠. 원인은 소표본·제외 경계 판단·도메인 갭이고, 다음은 50~100편으로 평가셋을
> 넓혀 재측정한 뒤 그래도 미달이면 레이아웃 모델만 파인튜닝하는 겁니다. 저희는 측정 없이 모델을
> 키우지 않는다는 원칙을 지키고 있습니다."

이 문구는 약점(레이아웃 미달)을 먼저 인정하고 원인·계획으로 신뢰를 회복하는 순서를 따른다 —
`EVALUATION_NOTES_AND_QA.md` §2의 프레이밍 규칙과 동일하다.

## 2. 씬별 콘티

### Scene 1 — 오프닝 (0:00–0:30)

- **화면**: `PRESENTATION.md` 1번 슬라이드(타이틀) 또는 동일 문구의 인트로 카드
- **자막**: `온프레미스 논문 분석 RAG`
- **내레이션**: "논문 PDF는 많지만 구조가 제각각이라 바로 검색할 수 없습니다. 이 프로젝트는 OCR로 논문을 구조화하고, 검수 과정을 눈으로 확인할 수 있게 만든 뒤, 검색 결과를 엑셀로 바로 활용하는 온프레미스 시스템입니다."

### Scene 2 — 아키텍처 소개 (0:30–1:20)

- **화면**: `assets/architecture-system.svg` → `assets/architecture-ai.svg` 순서로 전환(각 20초 내외)
- **내레이션**: "시스템은 세 부분으로 나뉩니다. 사용자는 업로드와 검색 두 경로로 들어오고, 요청은 FastAPI가 받아 PostgreSQL과 pgvector, Celery 워커로 넘깁니다. 문서 처리는 레이아웃 검출, OCR, LLM 정제·요약, 임베딩 순서로 이어지는 단일 파이프라인입니다."
- **자막 포인트**: "PP-DocLayout-M → PP-OCRv5 → Qwen2.5-7B → BGE-M3" (화면 하단 자막으로 한 줄)

### Scene 3 — 데모 ① 논문 등록·검수 (1:20–2:50)

- **화면**: 브라우저 화면 녹화, `/documents/{id}/viewer?editable=true` 실제 검수 화면
- **진행 순서**(각 15~20초):
  1. PDF 업로드 화면 → 레이아웃 검출 결과(블록 박스) 표시
  2. 블록 클릭 → OCR 텍스트 확인, 신뢰도 낮은 블록 강조
  3. "영역별 OCR 실행" → "OCR 검수 완료" 버튼 클릭 → phase가 `ready_to_ingest`로 바뀌는 화면
- **내레이션**: "논문을 올리면 페이지 전체가 이미지로 변환되고, 레이아웃 모델이 제목·저자·본문·표 영역을 자동으로 구분합니다. 검수자는 화면에서 박스와 OCR 텍스트를 직접 확인하고 승인하거나 고칠 수 있습니다."
- **주의**: 실제 오류 사례(예: 낮은 신뢰도 블록)를 하나 보여주면 "검수가 실제로 필요한 이유"가 더 잘 전달됨

### Scene 4 — 데모 ② 검색·엑셀 다운로드 (2:50–4:00)

- **화면**: Streamlit 검색 UI(`:8501`) 화면 녹화
- **진행 순서**(각 15~20초):
  1. 자연어 질의 입력 → 정확 키워드 매칭 결과(대표 논문 + 연관 논문) 표시
  2. 각 논문 카드 아래 "왜 이 논문인지" 관련도 설명(파란 info 박스) 강조 — 근거 단락 기반 생성 결과
  3. (선택) 매칭 실패 예시 → 유사 키워드 Top-3 제안 → 사용자 선택
  4. "엑셀 다운로드" 클릭 → 받은 xlsx 파일에서 시트 탭(검색결과요약/대표논문/단락/관련도 설명 등) 스크롤
- **내레이션**: "검색은 매 질의를 LLM으로 해석해 핵심 키워드를 뽑고, 정확한 키워드 매칭을 먼저 시도합니다. 실패하면 벡터 유사 키워드를 제안해 사용자가 고르게 합니다. 대표·연관 논문이 정해지면, 각 논문의 근거 단락을 바탕으로 '왜 이 논문이 질의와 관련 있는지'를 한두 문장으로 생성해 함께 보여줍니다 — 이 근거 생성 단계가 이 시스템을 단순 검색이 아닌 RAG로 만드는 부분입니다. 최종 결과는 원문·요약·표 데이터·관련도 설명을 담은 엑셀(최대 9시트)로 내려받습니다."

### Scene 5 — 검증 현황 · 다음 계획 (4:00–4:40)

- **화면**: `PRESENTATION.md`의 "형상관리 남은 공백" / "현재와 목표·우선순위" 슬라이드
- **내레이션**: "현재까지 자동화 테스트 188개가 통과했고, 모델이 실제로 기동해 응답하는 것까지 확인했습니다. 다만 실제 논문에 대한 OCR 정확도와 레이아웃 인식률은 아직 수치로 측정하지 않았고, 지금 15편 평가셋 라벨링을 진행 중입니다. 측정 없이 모델을 키우지 않는다는 원칙으로, 기준 미달 항목만 골라 개선할 계획입니다."

### Scene 6 — 클로징 (4:40–5:00)

- **화면**: 타이틀 슬라이드 재사용 + 팀/발표자 정보
- **내레이션**: "감사합니다."

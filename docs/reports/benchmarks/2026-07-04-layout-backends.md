# 레이아웃 백엔드 비교 — simple vs docling (2026-07-04)

합성 PDF 2종(WSL2, CPU)으로 두 백엔드를 실측 비교했다. 실제 논문 PDF 벤치마크(Phase 0, 평가셋 50편)의
축소 선행 실험이다.

## 입력

| 파일 | 구성 |
| --- | --- |
| `sample.pdf` (1p) | 제목·저자·서론 6줄·References |
| `rich_sample.pdf` (2p) | 제목·저자·초록·본문 2섹션·**괘선 표(4×3)**·표/그림 캡션·References 2건 |

## 결과 (rich_sample.pdf 기준, --skip-llm --dry-run)

| 항목 | simple | docling (수정 전) | docling (수정 후) |
| --- | ---: | ---: | ---: |
| 단락 수 | 3 | 8 | 8 |
| 표 추출 | 0 | 0 | **1** |
| 참고문헌 제외 | ✅ | ❌ (타입 유실) | ✅ |
| 처리 시간 | 0.5s | 12.5s | 13.2s |

## 발견 사항

1. **어댑터 결함 발견·수정**: 초기 docling 어댑터가 `export_to_markdown()` 통짜 텍스트를 쪼개는
   방식이라 표·섹션헤더·참고문헌 블록 타입이 전부 유실됐다. `iterate_items()` 기반 12블록 매핑으로
   재작성 후 표 추출·참고문헌 제외가 정상 동작 (커밋 `fix: docling 어댑터를 정식 아이템 매핑으로 재작성`).
2. **단락 분해 품질**: docling이 섹션 구조를 보존해 단락을 더 세밀하게 분리(8 vs 3).
   simple은 페이지 텍스트 블록 휴리스틱이라 병합이 과하다.
3. **속도**: docling은 CPU에서 페이지당 ~6초 수준(최초 실행 시 모델 다운로드 별도, `~/.cache`).
   대량 수집은 야간 배치 전제(DESIGN §9)와 부합.

## 추가 관찰 (실 스택 종단 실행에서)

- 합성 PDF의 제목이 docling 레이아웃 모델에서 title 라벨로 분류되지 않아 논문 제목이
  파일명으로 폴백됨 — 실제 논문 조판에서는 개선될 것으로 예상되나 Phase 0 평가셋에서
  제목 추출 정확도를 별도 항목으로 측정할 것
- 표 캡션(table_caption)과 표 본체의 연결(table_title 매핑)이 아직 없음 — 후속 개선 항목

## 결론·후속

- 디지털 PDF 기본 백엔드로 docling 사용 가능 상태 확인. **실제 논문 PDF 10편** 비교(2단 조판·스캔본
  포함)로 Phase 0 벤치마크를 진행해야 최종 확정 (DESIGN §6 지표: mAP/CER/TEDS/E2E F1).
- 표 구조 정밀도(TEDS)는 합성 표 1건으로는 판단 불가 — 평가셋에서 측정.

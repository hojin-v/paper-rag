# 로컬 LLM CPU 실측 — Qwen2.5 1.5B vs 7B (2026-07-04)

개발 머신(WSL2, CPU 추론, Ollama 컨테이너)에서 단락 정제·요약·키워드 JSON 생성
(`enrich_paragraph`) 1건을 실측했다. 운영 서버(16코어/64GB) 사전 참고용이며,
7B 측정은 1.5B 수집 배치와 CPU 경합 상태에서 수행되어 보수적 수치다.

## 결과

| 항목 | qwen2.5:1.5b-q4_K_M | qwen2.5:7b-q4_K_M |
| --- | --- | --- |
| 지연 (모델 로드 포함) | 219.6s (첫 호출) | **900s 타임아웃 ×2회 → Passthrough 폴백** |
| JSON 스키마 준수 | ✅ | 측정 불가 (생성 미완료) |
| 정제 품질 | ⚠️ 영한 혼용 오류 ("스mart factory environment에서…") | 측정 불가 |
| 요약 품질 | ⚠️ 원문에 없는 표현 삽입 ("디지털 시대의 공장") | 측정 불가 |
| 키워드 | 3개 추출, 대체로 적절 | 측정 불가 |

폴백 동작 확인: 7B 타임아웃 2회 후 파이프라인이 중단 없이 Passthrough(원문 유지)로
계속 진행됨 — 설계된 열화 경로가 실환경에서 정상 작동.

## 판단

1. **개발 머신에서는 1.5B로 기능 검증만** 수행한다. 1.5B의 정제·요약 품질(혼용·의역 오류)은
   운영 투입 불가 수준 — DESIGN §6의 "프롬프트 최적화 → 미달 시 파인튜닝" 게이트를 7B 기준으로
   운영 서버(또는 GPU)에서 측정해야 한다.
2. 7B CPU 추론은 설계 가정(논문 1편 15~30분, 야간 배치)에 들어오려면 16코어급 서버가 필요하다.
   개발 머신 기준으로는 시간당 단락 수 건 수준으로 비현실적.
3. LLM 타임아웃 기본값은 단락당 최악 케이스를 고려해 운영 서버 실측 후 재보정한다.
   (현재 폴백이 안전하게 동작하므로 파이프라인 전체가 막히지는 않음)

## 후속

- 운영급 CPU 서버 또는 GPU에서 7B 품질·지연 재측정 → 품질 합격선(DESIGN §6 표) 판정
- 1.5B 결과가 섞인 이번 실측 DB 데이터는 품질 판단에 사용하지 않는다 (기능 검증 전용)
